## คำอธิบายเกม
King-of-Fruit เป็นเกมจับคู่ภาพผลไม้ที่ท้าทายความจำและสมาธิของผู้เล่น โดยผู้เล่นจะต้องค้นหาและจับคู่ผลไม้ที่เหมือนกันให้ครบ 3 ใบภายในเวลาที่กำหนด เกมนี้ได้รับแรงบันดาลใจจากเกมจับคู่ภาพแบบคลาสสิก แต่เพิ่มความท้าทายด้วยการต้องจับคู่ให้ครบ 3 ใบแทนที่จะเป็นแค่ 2 ใบ ซึ่งทำให้เกมมีความซับซ้อนและน่าตื่นเต้นมากยิ่งขึ้น

เกมมีหลายระดับความยากให้เลือกเล่น โดยในแต่ละระดับจะมีจำนวนการ์ดที่แตกต่างกัน ยิ่งระดับสูงจำนวนการ์ดก็ยิ่งเพิ่มขึ้น และเวลาที่ให้ก็มีจำกัดมากขึ้น ผู้เล่นต้องอาศัยความจำที่ดีในการจดจำตำแหน่งของผลไม้แต่ละชนิด และใช้กลยุทธ์ในการเปิดการ์ดเพื่อให้สามารถจับคู่ได้อย่างรวดเร็ว เกมนี้เหมาะสำหรับทุกวัย ไม่ว่าจะเป็นเด็กที่ต้องการฝึกทักษะการจำ หรือผู้ใหญ่ที่ต้องการผ่อนคลายและฝึกสมองไปพร้อมกัน

## ฟีเจอร์หลักของเกม

### ระบบเกมเพลย์
- **จับคู่พิเศษ 3 ใบ**: กลไกการเล่นที่ท้าทายกว่าเกมจับคู่ทั่วไป โดยต้องหาผลไม้เหมือนกัน 3 ใบในคราวเดียว
- **ด่านที่หลากหลาย**: แต่ละด่านมีชื่อเฉพาะและความยากที่เพิ่มขึ้นเรื่อยๆ ตั้งแต่ "ทุ่งผลไม้" ไปจนถึง "สวรรค์ผลไม้"
- **ระบบเวลาจำกัด**: แต่ละด่านมีเวลาที่กำหนด (60-100 วินาที) เพื่อเพิ่มความตื่นเต้น
- **หลอดเวลาที่ฉลาด**: แสดงเวลาที่เหลืออย่างชัดเจน พร้อมเปลี่ยนสีจากเขียว → เหลือง → แดง ตามเวลาที่เหลือ
- **ระบบปลดล็อคด่าน**: ผู้เล่นต้องผ่านด่านปัจจุบันก่อนจึงจะปลดล็อคด่านถัดไปได้ สร้างแรงจูงใจในการเล่นต่อเนื่อง
- **ความยากที่เพิ่มขึ้น**: เริ่มจาก 7 ชุดการ์ด (21 ใบ) ในด่านแรก ไปจนถึง 35 ชุดการ์ด (105 ใบ) ในด่านสุดท้าย
- **หน้าจอแสดงผล**: แสดงผลการเล่นหลังจบเกม พร้อมสถิติต่างๆ

### ระบบเสียงและการตั้งค่า
- **เพลงประกอบ (BGM)**: เพลงพื้นหลังที่วนซ้ำอัตโนมัติ สร้างบรรยากาศในการเล่น
- **เอฟเฟกต์เสียง (SFX)**: เสียงประกอบการกระทำต่างๆ ในเกม
- **ปรับระดับเสียงได้**: แยกปรับความดังของ BGM และ SFX ได้อิสระ
- **ระบบ Mute**: สามารถปิดเสียงทั้งหมดได้ทันทีเมื่อต้องการความเงียบ

### ระบบ UI/UX ที่ใส่ใจ
- **ป้องกันการกดทะลุ**: การ์ดที่ถูกวางทับกันจะไม่สามารถกดการ์ดด้านล่างได้ ป้องกันการกดผิด
- **การออกแบบที่สวยงาม**: ใช้ฟอนต์น่ารัก มีเงา มุมมน และภาพสีสันสดใส
- **หน้าจอครบครัน**: แบ่งเป็น Start Screen, Level Select, Game Screen, Result Screen และ Settings Screen
- **การนำทางที่ลื่นไหล**: สามารถเข้าถึงฟังก์ชันต่างๆ ได้ง่าย และกลับไปหน้าเมนูได้ทุกเมื่อ

## 🏗️ ภาพรวมของโปรแกรม (Program Overview)

**King-of-Fruit** ถูกออกแบบโครงสร้างแบบ **Modular Architecture** แยกการทำงานออกเป็นสัดส่วน เพื่อให้ง่ายต่อการดูแลรักษาและเพิ่มเติมฟีเจอร์ในอนาคต

### 🗺️ Architecture Diagram
```text
King-of-Fruit/
│
├── main.py                  # จุดเริ่มต้นของแอปพลิเคชัน และจัดการ Global State
├── assets/                  # 🎨 แหล่งรวมทรัพยากร (รูปภาพ, เสียง, ฟอนต์)
│   ├── fonts/               # ไฟล์ฟอนต์ (.ttf)
│   ├── images/              # ภาพพื้นหลัง, รูปผลไม้, ไอคอน (.png)
│   └── sounds/              # เสียงดนตรี (BGM) และเอฟเฟกต์ (SFX)
│
└── screens/                 # 🖥️ ไฟล์หน้าจอต่างๆ ของเกม
    ├── start_screen.py        # หน้าเมนูหลัก (Start Menu)
    ├── level_select_screen.py # หน้าเลือกด่าน (Level Selection)
    ├── game_screen.py         # หน้าจอขณะเล่นเกมเพลย์ (Gameplay Core)
    ├── result_screen.py       # หน้าจอแสดงผลลัพธ์เมื่อจบเกม (Result/Score)
    └── settings_screen.py     # หน้าต่างสำหรับการตั้งค่าระบบเสียง (Settings)
```

### 📂 โครงสร้าง 3 ส่วนหลัก

1. **`main.py`**
   - ทำหน้าที่ประกาศ **Class App (`FruitMatchApp`)** ซึ่งเป็นจุด Entry Point ของตัวเกม
   - โหลด `ScreenManager` และทยอยเพิ่มหน้าจอ (`screens`) เข้าไประบบจัดการ
   - เก็บข้อมูล **Global State** ที่แชร์อ้างอิงตรงกันทุกจอ (ดูกล่อง Global State ด้านล่าง)

2. **`screens/`** (ส่วนหน้าจอผู้ใช้งาน)
   - **`start_screen.py`**: ต้อนรับผู้เล่น มีปุ่มเริ่มเกม, เลือกด่าน, และปุ่มตั้งค่า
   - **`level_select_screen.py`**: แสดงรายการ Level 1-5 พร้อมระบบสัญลักษณ์แม่กุญแจ หากยังไม่ปลดล็อคด่านถัดไปจะไม่สามารถเข้าเล่นได้
   - **`game_screen.py`**: แกนหลักของเกมเพลย์ สร้าง grid การ์ด วางการ์ดแบบสุ่ม จับเวลา ควบคุมการกดการ์ด (จับคู่ทีละ 3 ใบ) ตรวจสอบเงื่อนไขการชนะ/แพ้
   - **`result_screen.py`**: แสดงผลการเล่นหลังหมดเวลา หรือคู่ครบแล้ว ประกอบด้วยคะแนน (ดาว 1-3 ดวง) และสรุปความแม่นยำ
   - **`settings_screen.py`**: ให้ผู้เล่นปรับความดัง BGM/SFX และเปิด-ปิดเสียงทั้งหมด

3. **`assets/`**
   - แย่งหมวดหมู่ชัดเจน ช่วยให้การเรียก `source='assets/...'` ทำได้ง่าย ไม่สับสนกับซอร์สโค้ด

### ⚙️ Global State ที่เก็บใน App
โปรเจกต์นี้ใช้ `App.get_running_app()` เพื่อแชร์ข้อมูลสถานะระหว่าง Screen โดยไม่จำเป็นต้องผ่าน Parameter ไปมาให้วุ่นวาย ตัวแปรหลักๆ ได้แก่:
- `app.unlocked_levels` 📈: บันทึกว่าด่านไหนบ้างที่ผู้เล่นเคลียร์ผ่านแล้ว (เช่น `[1, 2]` หมายถึงด่าน 1 และ 2 ปลดล็อคแล้ว)
- `app.bgm_volume` & `app.sfx_volume` 🔉: เปอร์เซ็นต์ความดังเสียง
- `app.is_muted` 🔇: สถานะการปิดเสียง
- `app.bgm` 🎵: Object เก็บช่องสัญญาณเสียงเพลงพื้นหลัง เพื่อปรับ Volume ได้ทันที
